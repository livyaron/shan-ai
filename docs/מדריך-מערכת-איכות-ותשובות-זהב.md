# 📘 מדריך משתמש — Shan-AI: שאלות, פידבק ותשובות זהב

מדריך זה מסביר איך לעבוד עם מערכת Shan-AI: לשאול שאלות, לתת פידבק שמשפר את המערכת, ולמנהלים — איך לבנות "תשובות זהב" שמלמדות את המערכת מהי התשובה הנכונה.

---

## 1. מה המערכת עושה?

Shan-AI עונה על שאלות לגבי פרויקטים של תחנות משנה (מנהל פרויקט, שלב, תאריך חישמול, סיכונים וכו') — דרך טלגרם או דרך הדשבורד. בנוסף, יש לה **מנגנון איכות** שבודק כל הזמן עד כמה התשובות נכונות, ומשתפר מהפידבק שלכם.

הלולאה פשוטה:

```
אתם שואלים → המערכת עונה → אתם נותנים 👍/👎 → המערכת לומדת
מנהלים מאשרים "תשובות זהב" → המערכת יודעת מה נכון → ציון האיכות נעשה אמין
```

---

## 2. לשאול שאלה (כל המשתמשים)

**בטלגרם:** כתבו את השאלה ישירות, או `‏/ask <שאלה>`.
דוגמאות:
- "מי המנהל של פרויקט בת ים?"
- "מתי תאריך החישמול של אשלים?"
- "מה השלב הנוכחי של חולה?"

**בדשבורד:** עמוד **💬 שאל**.

> 💡 טיפ: ככל שתנסחו מדויק (שם פרויקט מלא), כך התשובה תהיה טובה יותר.

---

## 3. לתת פידבק — הכי חשוב! (כל המשתמשים)

מתחת לכל תשובה בטלגרם יש שני כפתורים:

- **👍 מועיל** — התשובה נכונה ועזרה.
- **👎 לא מועיל** — התשובה שגויה או חסרה.

כשאתם לוחצים **👎**, המערכת שואלת **מה היה לא בסדר?** ונותנת שלוש אפשרויות בלחיצה אחת:

| כפתור | מתי לבחור |
|---|---|
| **פרויקט לא נכון** | המערכת ענתה על פרויקט אחר |
| **חסר מידע** | המידע קיים אבל לא הופיע בתשובה |
| **תשובה שגויה** | התשובה כוללת פרט לא נכון |

> ⚠️ הפידבק הזה הוא הדלק של המערכת. כל 👍/👎 משפר את הציון ומכוון את הצוות לאן לתקן. לוקח 2 שניות — אנא עשו את זה.

---

## 4. תשובות זהב — למנהלים בלבד

"תשובת זהב" = התשובה הנכונה והמאושרת לשאלה מסוימת. המערכת משתמשת בה כדי לבדוק את עצמה: אם התשובה האוטומטית תואמת לזהב → ✅, אם לא → ❌. ככל שיש יותר תשובות זהב, כך מדד האיכות אמין יותר.

יש **שתי דרכים** לבנות תשובות זהב:

### דרך א' — בטלגרם (מהיר, מהנייד)

1. שלחו `‏/gold` (זמין למנהלים בלבד).
2. המערכת מציגה שאלה אמיתית מהשטח + **הצעה** לתשובה, עם כפתורים:
   - **✅ אשר** — ההצעה נכונה, נשמרת כזהב.
   - **✏️ תקן** — ההצעה לא מדויקת; שלחו בהודעה את התשובה הנכונה והיא תישמר.
   - **⏭ דלג** — לא בטוחים / שאלה לא רלוונטית, עוברים לבאה.
   - **⏹ סיום** — מסיימים.
3. אחרי כל אישור, המערכת מציגה את השאלה הבאה בתור — אפשר לעבור על כמה שרוצים ברצף.

### דרך ב' — בדשבורד

עמוד **🧪 איכות AI ← אימות תשובות זהב** (`/dashboard/eval/curate`):
- טבלת **"מועמדים מהשטח"** — שאלות שחוזרות הרבה ועדיין אין להן זהב.
- לכל שאלה: **הצע תשובה** → עורכים אם צריך → שומרים.
- כפתור **"אישור כל תשובות ה-DB"** מאשר בבת אחת את כל השאלות שהתשובה שלהן נגזרת ישירות מהנתונים (מנהל/שלב/תאריך).

> 🎯 יעד מומלץ: לפחות 50 תשובות זהב. ככל שיש יותר — מדד האיכות מדויק יותר.

## 5. דשבורד האיכות — למנהלים

עמוד **🧪 איכות AI** (`/dashboard/quality`) מציג את תמונת האיכות:

| מה רואים | מה זה אומר |
|---|---|
| **מדידה חיה מצטברת** | המספר העיקרי והאמין: מתוך שאלות הזהב שנבדקו לאחרונה מול המערכת החיה — כמה נענו נכון, ומה הכיסוי (כמה נבדקו מתוך הכל). מתעדכן אוטומטית כל הזמן. |
| **אחוז הצלחה (שאלות ייחודיות)** | אותו רעיון אך לפי כל שאלה שונה פעם אחת (ללא ספירת חזרות). |
| **מגובי-gold** | כמה פסקי-דין נבדקו מול תשובת זהב אמיתית (אמין) מול ניחוש. |
| **סוגי כשל** | דירוג הסיבות לכשל: פרויקט לא נכון / חסר מידע / תשובה שגויה. מכוון לאן לתקן. |
| **שאלות נפוצות** | השאלות שחוזרות הכי הרבה + פסק הדין שלהן. |
| **שאלות שנכשלו** | רשימת הכשלים האחרונים — יעדי התיקון הבאים. |

### איך המדידה עובדת (אוטומטית)
המערכת **בודקת את עצמה כל הזמן** — אין צורך להריץ ידנית:
- כל 3 שעות רצה **מדידה חיה בקבוצה קטנה**: היא שואלת מחדש כמה שאלות זהב דרך המערכת האמיתית ובודקת אם התשובה נכונה. קבוצות קטנות = בלי עומס, מספר יציב (לא קופץ).
- הבדיקה **מבוססת-עובדות**: התשובה נחשבת נכונה אם היא מכילה את הפרויקט/הנתונים הנכונים — לא משנה הניסוח. כך תשובה נכונה לא "נופלת" בגלל פורמט.
- כל שאלה מקבלת פסק-דין עדכני; הדשבורד מציג את הממוצע המצטבר — **מספר אמין שיציב לאורך זמן**.

**כפתורי פעולה (ידני, לא חובה — האוטומציה עושה את רוב העבודה):**
- **🌱 זריעת gold מ-DB** — יוצר אוטומטית תשובות זהב לכל שאלה שהתשובה שלה נגזרת ישירות מהנתונים. התחילו מכאן.
- **📡 מדידה חיה (gold)** — מריץ מיד בדיקה חיה (אם לא רוצים לחכות למחזור האוטומטי).
- **⚖️ שפוט שאלות אחרונות** — סיווג שאלות מהשטח שעדיין לא נבדקו.
- **♻️ שיפוט מחדש** — בודק מחדש אחרי שהוספתם תשובות זהב.

## 6. מילון מונחים

| מונח | פירוש |
|---|---|
| **תשובת זהב (gold)** | התשובה הנכונה המאושרת ע"י מנהל — אמת המידה. |
| **PASS / ✅** | התשובה האוטומטית תאמה לאמת. |
| **FAIL / ❌** | התשובה לא תאמה. |
| **מגובה-gold** | הציון נקבע מול תשובת זהב אמיתית (אמין), לא מול ניחוש. |
| **שאלה ייחודית** | שאלה אחת, ללא ספירת חזרות. |
| **מדידה חיה** | המערכת שואלת מחדש את שאלות הזהב דרך המנוע האמיתי ובודקת — משקף את ההתנהגות הנוכחית, לא היסטוריה. |
| **בדיקה מבוססת-עובדות** | תשובה נכונה = מכילה את הפרויקט/הנתון הנכון, בלי תלות בניסוח. |
| **כיסוי** | כמה שאלות זהב נבדקו לאחרונה מתוך הסך הכל. |

## 7. תהליך מומלץ למנהל (סיכום)

1. היכנסו ל-**🧪 איכות AI** בדשבורד.
2. לחצו **🌱 זריעת gold מ-DB** — מאכלס אוטומטית את הקל.
3. השלימו ידנית עוד תשובות זהב דרך **`/gold` בטלגרם** או **אימות תשובות זהב** בדשבורד.
4. זהו — מכאן המערכת **מודדת את עצמה אוטומטית** (מדידה חיה כל 3 שעות). פשוט חוזרים לדשבורד ורואים את **המדידה החיה המצטברת**.
5. הסתכלו על **סוגי כשל** ו**שאלות שנכשלו** — אלה יעדי התיקון הבאים לצוות הפיתוח.
6. עודדו את כל המשתמשים ללחוץ **👍/👎** על תשובות — זה מה שמזין את הכל.
