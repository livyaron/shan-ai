3. עמוד "שאל"

הצ׳אט הוא נקודת המוצא לכמעט הכל: שאלה על פרויקט, חיפוש קובץ, או תיעוד החלטה. המערכת מזהה אוטומטית מה הקלדת ומפנה לזרימה הנכונה.

איך השאלה מסתיימת בתשובה

שאלה רגילה

  1. הקלד/י שאלה בעברית. למשל: "מי המנהל של פרויקט בת ים?".
  2. לחיצה על שלח — תשובה תוך 1–4 שניות.
  3. למטה תופיע התשובה + ציון מקור (📁 קבצים, 📂 פרויקטים, 📋 החלטות).
  4. אם התשובה נכונה — לחיצה על 👍.
  5. אם שגויה — לחיצה על 👎, הקלד/י את התשובה הנכונה — המערכת לומדת מיידית.

  • "איזה פרויקטים בעיכוב?"
  • "מה ההחלטות האחרונות שלי?"

תיעוד החלטה

אם הקלדת משפט שנשמע כמו החלטה (לדוגמה: "אני מחליט לעצור את העבודות בבת ים"), המערכת תזהה את זה אוטומטית ותפתח לך טופס אישור עם:

  • סוג ההחלטה (INFO / NORMAL / CRITICAL / UNCERTAIN — ראה איור 3.2)
  • סיכום + פעולה מומלצת
  • סיכונים + הנחות
  • מדידות (האם ניתן למדוד את ההצלחה?)
  • טבלת RACI — שיבוץ מבצע, בעל סמכות, יועצים, נמענים

אחרי שאישרת — לחיצה על ✓ אשר ושלח שולחת את ההחלטה לאישור (אם קריטית) או ישירות לתיעוד.

4. RACI — מי עושה מה

כל החלטה מקבלת שיבוץ של 4 תפקידים. המערכת מציעה — את/ה מאשר/ת או משנה לפני שליחה.

כללים חשובים

  • חייב להיות בדיוק בעל סמכות אחד (A) — אחרת ההחלטה לא תישמר.
  • מבצעים (R) יכולים להיות כמה — צוות שמבצע את הפעולה.
  • יועצים (C) נשאלים לפני שהפעולה נעשית.
  • נמענים (I) מקבלים הודעה אחרי שהפעולה בוצעה.
